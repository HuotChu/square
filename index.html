SquareDB

SquareJS includes SquareDB (Structured QUery And RElational DataBase) which allows developers to store client-side data in-memory using a relational data store. Data can be retrieved from the store using structured queries nearly identical to SQL syntax. Possible uses include one-to-one mapping of backend data to the frontend, reuse of SQL queries from your PHP or backend code, mapping of data objects to relational tables, SQL storage on mobile platforms, and much more. SquareDB is also the perfect tool for SQL instructors. Students can write and execute SQL using only a browser - no backend database, no PHP, no outside dependencies at all! Need data binding as well? Check out the SquareJS "EventHub"...

EventHub

EventHub enhances objects (and therefor, models) with real JavaScript eventing. This creates an environment which works like DOM eventing, except that events are dispatched from JavaScript Object Nodes and can bubble up to parent object nodes. Data binding is as simple as attaching event handlers to the model the same way you would attach them to the DOM. i.e. Model.addEventListener('model.dataNode.change', callBackFunc); and if you want to bind that handler directly to a DOM node, simply pass that node as a third parameter to the handler during instantiation. i.e. Model.addEventListener('model.dataNode.change', callBackFunc, domNode);

Temple

I'd rather be writing JavaScript. Period. So when it came to creating a template engine the idea was simple, "make it an HTML file, make it validate, and make the syntax easy". Temple accomplishes all of these things in a super lightweight API that provides everything you need for DOM templating.

Request

Request supplies SquareJS with Promise-wrapped AJAX functionality.

Box

Box takes the place of a controller in Square. Use a box to bring together a template, model data, and event handlers for a convenient and logical way of generating views within an application.

Lobro

Lobro manages localStorage concerns like persisting and syncing the model data.
